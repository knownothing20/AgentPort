# Local Configuration

This directory contains example files plus local runtime configuration.
Real tokens, passwords, private keys, runtime state, logs, and generated files
are **NOT uploaded to Git** (see `.gitignore`).

## Configuration Files

### 1. `mcp-remote-agent.json` - Main Configuration

Copy from `mcp-remote-agent.example.json` in the project root:

```bash
cp mcp-remote-agent.example.json local/mcp-remote-agent.json
```

Then edit `local/mcp-remote-agent.json` and fill in your values:

| Variable | Description | Example |
|----------|-------------|---------|
| `remoteUrl` | Remote daemon address | `http://192.168.1.100:3183` |
| `authToken` | Client authentication token | `my-client-token-abc123` |
| `clientId` | Client identifier for audit logs | `my-workstation` |
| `skillDir` | Absolute path to this skill directory | `C:\Users\you\.workbuddy\skills\mcp-remote-agent` |
| `mcpConfigPath` | Path to your AI tool's MCP config | `C:\Users\you\.workbuddy\mcp.json` |

### 2. `connections.json` - Multi-Server Connections (Optional)

For dynamic connection switching between multiple servers:

```json
{
  "connections": [
    {
      "name": "server1",
      "description": "Production server",
      "url": "http://192.168.1.100:3183",
      "authToken": "token-for-server1",
      "clientId": "my-workstation"
    },
    {
      "name": "server2",
      "description": "Development server",
      "url": "http://192.168.1.200:3183",
      "authToken": "token-for-server2",
      "clientId": "my-workstation"
    }
  ],
  "default": "server1"
}
```

### 3. `server/.env` - Server Configuration (Auto-generated)

This file is auto-generated by `sync.cjs` from `mcp-remote-agent.json` variables. Do not edit manually.

Location: `local/server/.env` (created by `node sync.cjs`)

## Directory Structure

```
local/
├── config-guide.md             # This file

└── server/

```

## Install On Another Computer

From a fresh clone, create real config files from the examples:

```bash
cp local/connections.json.example local/connections.json
cp mcp-remote-agent.example.json local/mcp-remote-agent.json
```

If you are migrating from another computer, copy your private
`local/connections.json` and `local/mcp-remote-agent.json` through a secure
channel, then update local paths such as SSH private key locations.

## Security Notes

- **Never commit** real files such as `local/connections.json`,
  `local/mcp-remote-agent.json`, `local/server/.env`, or SSH private keys
- **Never share** your auth tokens
- Use strong, unique tokens for each client
- Rotate tokens regularly
